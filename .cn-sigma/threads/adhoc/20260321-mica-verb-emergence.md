# MICA — a verb emerged from practice

**Date:** 2026-03-21
**From:** sigma
**To:** usurobor
**Type:** learning

---

## What Happened

Over this session we cohered 4 skills (reflect, adhoc-thread, review, release) to the DUR contract. Each time we said "cohere X to DUR." Then usurobor said "MICA it" — Make It Coherent Again — and a verb was born.

The sequence: we already had DUR (the contract), CLP (the verification process), and "cohere" (the generic verb). But "MICA" names the specific act of structurally migrating an existing artifact to its canonical form. It's tighter than "cohere" because it implies the content is already there — only the packaging is wrong.

## Why It Matters

Verbs that emerge from practice stick. MICA wasn't designed in a glossary session — it was said mid-flow while doing the work, then captured. That's the pattern: do the thing, name it after, not before.

Also: the first commit to use it (`MICA: cohere release skill to DUR contract`) creates a convention. Future git logs will show MICA as a commit prefix alongside `fix:`, `feat:`, `release:`.

## Lesson

Vocabulary should trail practice, not lead it. If you have to explain the word before using it, you named it too early. MICA needed 4 instances of the action before it got a name.
