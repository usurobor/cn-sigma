# CDD Without GitHub PRs

**Date:** 2026-04-28
**Issue:** #283
**Status:** Filed, γ dispatched

## What happened

Operator identified that α still creates PRs and subscribes to them. Discussion surfaced:

1. **No one creates GH PRs** — the question was whether PRs are needed at all in `.cn` hub repos
2. **`.cdd/unreleased/{N}/` is the coordination surface** — all three roles (α/β/γ) write artifacts there during in-version work
3. **Hub threads are post-version only** — not for active branch work
4. **One dir per issue, flat filenames** — e.g. `alpha-closeout.md`, `beta-review.md`, `self-coherence.md`
5. **No numbered prefixes** — filesystem timestamps + git history carry ordering

## Path corrections

- Initially wrote `.cn/unreleased/<issue>/` — wrong. Correct path is `.cdd/unreleased/{N}/` per CDD.md §5.3a
- Initially wrote `{role}/{N}.md` (role-first) — corrected to `{N}/` (issue-first, role in filename)
- γ caught the path error in its quality gate audit before dispatch — the gate worked

## Scope

10 files across the CDD skill surface. Heaviest change is CDD.md §Tracking (~100 lines of PR polling mechanics to replace). All role algorithms (α §2.7, β step 3, γ §2.5) reference PRs.
