## Design decisions

### Triad structure
- α/β/γ are not agents — they are identities (personalities) of one agent
- Structure: dyad (α↔β isolated) + coordinator (γ sees both, does neither)
- TSC is the kernel identity model, not a CDD add-on
- Emerged from practice (cost optimization → coordinator → TSC recognition)

### CDD independence
- `.cdd/` at repo root, not `.cn/cdd/`. CDD doesn't require cnos.
- Identity: `alpha@cdd.cnos` — practice-namespaced, project-scoped
- No forge dependency. Git is the practice. GitHub is infrastructure.

### Protocol > execution model
- `.cdd/` artifacts define the protocol (what, when, who)
- Three persistent sessions, one switching process, or hybrid — all use the same protocol
- No STATE.json — artifact presence IS the state
- Ship Phase 1 (audit trail) first, grow from evidence

### Skill loading tiers
- Tier 1 (CDD lifecycle): always loaded
- Tier 2 (general eng): always loaded
- Tier 3 (issue-specific): named per issue
- Dispatch prompt is minimal: role + project + issue. Skill tells the rest.
