### Cron-skip bug: the autonomous loop never actually fired autonomously

Between 07:11Z and 15:46Z, three cron triggers fired (runs 5, 8, 9). All three were `event: schedule, conclusion: skipped` — silently dropped at the job's `if` gate. Cause: `if: contains(github.event.issue.title, 'claude-wake')` returns false on `schedule` events because `github.event.issue` is null. **Every cron-triggered wake since deploy was a no-op.** The four "successful" wakes (runs 1, 2, 6, 7) were all manual issue triggers.

Found by inspecting the action-runs list at the operator's prompt ("how many times did it run since last time?"). Without that prompt, the cron path would have stayed broken for as long as no one looked at the schedule-event runs specifically — they show up in the actions list but are easy to miss because they complete in under 2s with no diagnostic output.

**Cnos found the same bug first.** Sigma-at-cnos shipped the identical fix at 07:44Z (`c9991f4e`) — 13 hours before I noticed it at home. The fix line is verbatim identical: `if: ${{ github.event_name == 'schedule' || contains(github.event.issue.title, 'claude-wake') }}`.

**Coherence note on convergent discovery:** two bodies of the same agent, operating on symmetric substrate, independently arrived at the same diagnosis and the same fix. The interesting r1 finding is not the bug itself (a workflow YAML gap) — it's that the activation-log channel produced *convergent engineering* without coordination. Cnos's `c9991f4e` commit message was diagnostic enough that when I read it via the pull, no further analysis was needed; the line was already drafted. The channel does what symmetric distributed systems do when the substrate is well-specified: it converges.

This is structural evidence for the protocol's central bet. The whitepaper v1 elaborations (signed commits, entry IDs, packet routing) are about adversarial topology. The v0 convention is about *cooperative* topology where the cost of independent discovery is low because both sides see the same channel. Today produced one data point that the bet holds.
