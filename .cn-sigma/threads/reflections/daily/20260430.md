## Key learnings

### 1. Sequential dispatch model (→ adhoc thread)
Nested γ→α/β is fragile (no output, SIGTERM cascades, memory pressure). δ dispatches all roles one at a time. γ produces prompts, δ executes. Codified in gamma/SKILL.md and operator/SKILL.md.

### 2. β needs Bash (→ adhoc thread)
`--allowedTools "Read,Write"` is too restrictive for β. Role boundary enforcement works via skill text, not tool scoping. beta/SKILL.md now explicitly constrains β from release actions. Verified in cycle #84.

### 3. Minimal prompts work
Role skills contain all instructions. Prompt is just:
```
You are {role}. Project: cnos.
Load {role}/SKILL.md and follow its load order.
Issue: gh issue view N --json title,body,state,comments
```

### 4. α may exit without committing
Cycle #84 fix round: α described the fix but didn't apply it. δ applied mechanically. #295 needs artifact verification after dispatch.

### 5. β role boundary enforcement works
After beta/SKILL.md was updated, β in cycle #84 merged and wrote close-out but did NOT tag/release. Skill-text enforcement is sufficient when the skill is clear.

### 6. Operator review improves shipped quality
eng/troubleshoot went through 4 operator review findings (escalation path, flexible triage, sources, MCA→corrective action) plus 2 consistency patches. The final skill is materially better than α's first output.
