### AC15a block to post (verbatim)

````markdown
## AC15a — canonical operator-surface format

Cohere terminal reports surfaced to operator (via wake notify, coherer session, or downstream surface) follow the canonical shape:

```
🔔 <sha> — exp: <forecast>, act: <actual> [<verdict|class>]
```

Collapsed heartbeat form acceptable:

```
🔔 <sha> — rest [heartbeat]
```

Field semantics:

- **`sha`** — short SHA (8 chars) of the commit that produced the report. Anchors the report to a git-addressable point.
- **`forecast`** — what the prior wake (or coherer session) said would happen by this report's horizon. Per AC17. If `no_prior_forecast`, the field may be omitted or set to `none`.
- **`actual`** — what actually happened. Short phrase (`rest`, `bumpt heartbeat`, `directive 1 still blocked`, `cohere issue opened: #444`, etc.).
- **`verdict`** — per AC17: `met` / `partial` / `missed` / `exceeded` / `superseded` / `no_prior_forecast`. Use when prior forecast horizon has elapsed.
- **`class`** — per AC8/AC14: `heartbeat` / `substantive` / `degraded:<reason>` / `failed:<reason>`. Use when no prior forecast applies.

The bracket carries either `verdict` or `class`, depending on whether a prior forecast is being closed. Both may appear when relevant: `[met, substantive]`.

Material deltas expand inline below the line as a one-paragraph summary. Routine outcomes hold to one line.

### Two surfaces aligned

- **Wake → coherer-watch PR comment** (the bot-posted notification): adopts the canonical shape. Lets the coherer session parse cheaply on receipt.
- **Coherer → operator** (the long-running session's surface): canonical. Replaces the free-form commit-subject-as-PR-comment shape that current wake yamls emit.

### Examples (operational evidence from the cn-sigma session 2026-06-04..2026-06-09)

```
🔔 b76ca477 — rest [heartbeat]
🔔 8a43293e — exp: pending, act: directive 2 (cohere-issue) DONE ✓ [met, substantive]
🔔 558e7f80 — exp: rest, act: cnos pushed with A7 flag (concurrent wake collision) [missed, degraded:wake_collision]
🔔 3fb2bfa9 — exp: pending, act: directive 4 (convergence amendment) DONE ✓ [met, substantive]
```

### Derivation

The format emerged organically during the operator-coherer session 2026-06-04 → 2026-06-09 as the practical compression for high-throughput wake notifications (one comment every ~15 minutes from the cn-sigma wake). It works because:

1. The 🔔 prefix marks "wake event" — distinguishes autonomous-loop output from operator-dialogue.
2. SHA-first lets coherer/operator cross-reference git state without parsing.
3. `exp / act` structure embeds AC17 trajectory semantics in the surface itself.
4. `rest` collapses the common case (heartbeat) without ceremony.
5. Bracket carries the protocol classification (verdict or class) compactly.
6. Inline expansion is allowed when material — the format doesn't force loss of detail.

### Citation

This AC derives from the operator-coherer dialog at `cn-sigma:threads/activations/cnos/20260604.md` → `20260609.md`. Specifically the session in which operator named the format and asked "shall we make it official?" — answered yes, dispatched as Directive 5.
````
