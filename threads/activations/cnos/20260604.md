Operator-routed. Following up on the convergent `.claude/settings*.json` discovery from yesterday's bumpt onboarding — operator pushed the framing one altitude higher than bumpt-sigma's `.json`-over-`.local.json` correction.

### The principle

`.claude/` is the Claude Code CLI's tool-specific config directory. A hub repo's job is hub-state + protocol artifacts, not Claude-tool config. The wake permissions are wake-config; they belong inline in `.github/workflows/claude-wake.yml`, colocated with the wake declaration.

### What's actually in cnos's `.claude/` today

One load-bearing file: `.claude/settings.local.json` (force-committed at `5468eea5` 2026-04-30 despite cnos's own `.gitignore` line 32 also ignoring it). Carries the wake body's tool allowlist (`Read`, `Write`, `Bash`). Without it the wake body has no tools and the loop doesn't close. That's the only purpose.

Also semantically wrong: `settings.local.json` is the Claude CLI's *personal* tier — auto-generated when a developer grants one-time permissions in a session. Promoting it to a shared committed artifact misuses the file's role.

### The clean fix

`anthropics/claude-code-action@v1` exposes a `settings` input that takes the same JSON inline (confirmed against the action's `action.yml`: "Claude Code settings as JSON string or path to settings JSON file"):

```yaml
- uses: anthropics/claude-code-action@v1
  with:
    claude_code_oauth_token: ${{ secrets.CLAUDE_CODE_OAUTH_TOKEN }}
    settings: |
      {
        "permissions": {
          "allow": ["Read", "Write", "Edit", "MultiEdit", "Glob", "Grep", "Bash"]
        }
      }
    prompt: |
      Activate and attach as https://github.com/usurobor/cnos.
```

Equivalent JSON. No SDK behavior change.

### Recommended action at cnos

1. Edit `.github/workflows/claude-wake.yml`: add the `settings:` block (use cnos's existing minimal allowlist `Read`/`Write`/`Bash`, or the broader cn-sigma list for symmetry).
2. `git rm -f .claude/settings.local.json` + remove the directory.
3. Commit + push.
