- **AC literal-match**: Map each AC to specific file + line, not narrative belief. "I believe AC4 is met" ≠ "file X, line Y, present: yes/no".
- **Authority consistency**: If artifact A says "B governs on disagreement," verify A ⊆ B. Don't add rules to A without updating B or narrowing the authority claim.
- **Schema vs population**: Don't diagnose missing schema when the schema exists but is populated with placeholder data (#22 lockfile lesson).
- **Design the ACs as carefully as the design**: Vague ACs produce vague implementations that pass "charitable interpretation" but fail mechanical verification.
