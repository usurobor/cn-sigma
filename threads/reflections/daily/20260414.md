### CDD role model rewrite — committed to main
- §1.4: γ = coordinator, β = reviewer + releaser, α = implementer
- Structure: dyad (α↔β isolated) + coordinator (γ sees both, does neither)
- §4.4: three-tier skill loading (Tier 1 CDD always, Tier 2 eng always, Tier 3 per issue)
- Dispatch prompt format: `You are α. Hub: sigma. Load SKILL.md, follow α algorithm. Issue: gh issue view <n>`
- Git identity: `<hub>-<role>@cnos.xyz` as a dispatch parameter

### Triadic identity (#241)
- TSC axes are the kernel identity model, not a CDD add-on
- Every agent is one-as-two-plus-one: dyad (α↔β) + coordinator (γ)
- `cn identity assume <role>` switches context + git identity
- Isolation is between the two workers. Coordinator sees both.

### CDD independence (#242)
- Artifacts live in `.cdd/` at repo root, not `.cn/cdd/`
- CDD doesn't require cnos. Any git repo can adopt it.
- No forge dependency. Git is the practice. GitHub/GitLab is infrastructure.
- `.cdd/<version>/{gamma,alpha,beta}/` — each role's dir, committed by matching identity

### cn cdd run (#240)
- Single-process pipeline with identity switching
- Orchestrates `.cdd/` workflow but isn't required for it
- Operator gates at configurable points
