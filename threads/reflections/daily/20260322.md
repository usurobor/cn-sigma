## β — What did I learn?

- **I merged without CI.** Read the code, approved, merged — never built or tested. Four compile errors shipped to main. The branch had 14 clean commits and read perfectly. Code reading is necessary but not sufficient. Always build before merge.
- **Version coherence bit us again.** `cn_lib.ml` vs `cn.json` — two sources of truth for one version number. This is exactly #22 and it cost a CI cycle to diagnose. The irony of shipping version-incoherence in a release about architectural coherence.
- **I proposed solutions to solved problems.** Suggested body-embedded delimiter blocks and provider tool-call schemas for op serialization. usurobor pointed out frontmatter JSON already had schema validation, denial receipts, and structural op/prose separation. The "10% gap" I was solving would have been a regression. Lesson: verify the existing system before proposing replacements.
- **The band-aid pattern is a reliable signal.** When you can arrange recent fixes in a causal chain, the root is structural. MCI → MCA → design → verify collapse is the right response, not "fix the next one faster."
- **Serialization format and execution model are coupled.** You can't borrow tool-call protocol for formatting while keeping post-call governance — the protocol implies the provider drives the loop.
