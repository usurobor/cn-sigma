## γ (What did I learn?)

- **New role skills need live testing before trust.** The δ skill had three bugs discovered in first use. Writing the skill is necessary; testing it against a real cycle is what makes it durable.
- **Daily decision-capture is a distinct failure class from behavioral reflection.** The reflect skill covered "what happened" but not "what was decided and why." These are different because decisions propagate across sessions as inherited authority — behavioral reflections don't.
- **Reachability is the third leg of polling.** Query + wake-up were named; reachability wasn't. β's troubleshooting report showed the exact shape: MCP tools can't run in Monitor, sandbox blocks api.github.com. The fix generalizes.
- **CTB v0.1 is the first spec that names what practice already built.** The CDD triad was the prototype runtime; the spec is what the runtime asked for. The `calls`/`calls_dynamic` split, scope-as-first-class, plan-as-data — all were already running in practice before the spec named them. The spec's job is to make them enforceable, not to invent them.
- **δ→γ signal is the missing handoff.** γ can poll git-observable state but can't poll platform state changes (tag pushed, branches deleted, CI results). Without δ's explicit completion signal, γ stays in observation mode indefinitely. The fix is small (one skill section) but the gap was invisible until the first live cycle.
- **δ is whole-to-whole composition, not a tetrad.** `COHERENCE-FOR-AGENTS.md` (written 2026-04-25) predicted this structure before δ was built: the triad stabilizes as one whole, the next participant creates a new one-as-two boundary with that whole. δ emerged from operational need the next day and landed exactly where the theory said. I initially described it as "triad becomes a tetrad" — counting parts — which is the exact failure the essay warns against ("the system grows by composing wholes, not by counting parts"). Theory available in the repo, not consulted until the operator pointed at the gap. See [adhoc thread](https://github.com/usurobor/cn-sigma/blob/main/threads/adhoc/20260426-delta-operator-role.md).
