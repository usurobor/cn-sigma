## Key findings

The structural compliance gate from wave 4 required an artifact that can't exist at check time. Self-application caught it within one cycle. Fix: `gamma-closeout.md` → `gamma-scaffold.md` in review/SKILL.md and beta/SKILL.md.

### Codex CLI model selection
- ChatGPT account auth blocks old model names (`o3`, `gpt-4.1`, `o4-mini`) — these don't exist in Codex model list
- Available: `gpt-5.5`, `gpt-5.4`, `gpt-5.4-mini`, `gpt-5.3-codex`, `gpt-5.3-codex-spark`
- Default is `gpt-5.5` (base, not Pro)
- Config flags: `-c model_reasoning_effort="xhigh"` `-c model_verbosity="high"`
- `gpt-5.3-codex-spark` (Pro-only) works — account is Pro tier

### Multi-agent issue refinement works
GPT and Claude converging on an issue body through adversarial rounds produces better issue packs than either alone. GPT tends to over-specify; Claude trims but sometimes removes needed structure; GPT restores with refinement. 3 rounds to convergence.

### γ adds overhead but catches bugs
The γ-included cycle took 4 dispatches (γ scaffold, α, β×3) vs wave 4's 2 per issue (α, β). But it produced the scaffold artifact that caught the #355 bug. Tradeoff is real.
