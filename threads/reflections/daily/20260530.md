## Sigma activation log convention v0 — shipped, then renamed

After a long YAGNI walk through messaging design (mail-dirs → sender-side namespace → file-per-message → append-to-log), the operator pointed at `cnos:docs/alpha/protocol/WHITEPAPER.md` v3.0.0 — Protocol v1 already specifies the append-to-log shape (union merge, trailing-newline rule, signed commits, entry IDs, `cn.json`). I had been re-deriving v1 from first principles. Same recurrence as the foundational-docs anti-pattern: read canonical source before reasoning.

Operator then converged on the minimum field convention for v0 (no signing, no envelopes, no entry IDs):

- `<foreign-repo>:.cn-sigma/log.md` — single writer = Sigma-at-foreign-repo
- `cn-sigma:threads/activations/<name>.md` — single writer = Sigma-at-home
- Cursors split: home-side `last_read_foreign_log` in `state/activations.md`; foreign-side inline log entry `Read home directives through cn-sigma@<sha>`.
- Entry format: `## YYYY-MM-DD — subject` + body. No frontmatter.
- Trust: push permission + single-writer + git history.
