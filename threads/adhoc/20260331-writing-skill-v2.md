# Writing Skill Rewrite

Date: 2026-03-31

## What changed

Writing skill went from a sentence-level style guide (cut filler, active voice, no hedging) to a document-architecture skill. Now operates at five levels: document → section → paragraph → sentence → word.

Key additions:
- governing question (name the file's job before writing)
- stable facts (say once, point elsewhere)
- one file one job (README/ARCHITECTURE/QUICKSTART split codified)
- Define → Unfold → Rules → Revision Pass algorithm
- final test checklist

## Why

The old version could fix a bad paragraph. The new version can fix a bad repo surface. Demonstrated live on tsc: three docs rewritten to one-job-each in the same session.

## Evidence

tsc README went from ~300 lines of mixed purpose to ~90 lines answering one question. ARCHITECTURE from ~120 to ~50. QUICKSTART from ~60 to ~30. No meaning lost.
