## Patterns

- **Matched-tag close is correct for XML block walk** — generic `</` matching fails on nested tags. Track the opening tag name, only close on `</tag_name>`.
- **5 rounds is too many for a P0 bugfix** — most findings were mechanical (duplicates, unused variables, test expectations). §2.1.3 should reduce this in future cycles.
- **Three reviewers converged** — all caught `<cn:ops>` (AC table forced it). Divergence was in exploration depth. Mechanical steps produce convergent output.
